# Contrib

The `contrib` directory contains supplemental examples, integrations, and recipes for building applications. Many
of these are contributions by other users who have found useful patterns, and code that may be beneficial for
others.

Content in this directory is intended to help users learn, and explore integrations to see how specific features
can be implemented.

---

# Contribution Rules

- Keep contributions focused and self-contained.
- Prefer readability and include comments where possible.
- Include a README or docstring header for non-trivial contributions.
- Avoid mixing unrelated concepts together.
- Keep code style consistent with the repository.
- Avoid heavy use of overriding internals, as these may change with versioning.
